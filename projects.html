Project 01
M.S. Thesis · RSCA Conference 2025

Predicting Conflicts & Analyzing Boycotts
Through Reddit Posts

A large-scale NLP study of online conflict discourse during the Israel-Palestine conflict — analyzing sentiment patterns, topic clusters, and behavioral triggers across a 5GB Reddit dataset from September 2023 to January 2025.

Python BERT NLP Sentiment Analysis Topic Modeling Reddit API Time-Series Analysis

Objective

To determine whether patterns in Reddit discourse could serve as leading indicators of consumer boycott activity, and to characterize the linguistic and emotional signature of escalating online conflict — with implications for brand risk monitoring and social media analytics.

Methods

Collected subreddit data via Reddit API. Applied BERT-based sentiment analysis with sentence-level splitting to handle model token limits on long-form comments (binary positive/negative — no neutral). Used LDA topic modeling to surface dominant themes. Time-series analysis tracked sentiment shifts relative to real-world conflict events and subreddit groups.

My Role

Sole researcher. Responsible for full pipeline design — data collection, preprocessing, model selection and tuning, temporal analysis, and visualizations. Designed the conference presentation independently. Presented at the 13th annual RSCA Conference (2025).

→

Key findings: 77% of comments were negative; negative comments averaged 60% longer, indicating deeper emotional engagement. Conflict events directly triggered multi-day negativity surges. Top posts consistently blamed targeted groups. Emotionally charged vocabulary ("terrorist," "crime") dominated negative word clouds. Anger propagated through high-engagement posts — offering a scalable framework for brand risk monitoring via social sentiment tracking.

Project 02
Big Data · Distributed Computing

Online Grocery Recommendation System

A scalable, personalized recommendation engine built on Apache Spark for a large-scale e-commerce grocery dataset — designed to improve product discovery through collaborative filtering at production-ready scale.

PySpark Spark SQL Spark MLlib ALS Algorithm Collaborative Filtering AWS
Objective

To build a personalized product recommendation system that improves shopping relevance at scale — reducing discovery friction in an online grocery platform while surfacing high-probability products for each user segment.

Methods

Implemented collaborative filtering using Spark MLlib's ALS algorithm on large-scale transaction data. Designed the full data pipeline in PySpark for distributed processing. Evaluated recommendation quality using RMSE and coverage metrics across user segments, comparing against a popularity-baseline approach.

My Role

Responsible for pipeline architecture, model selection, hyperparameter tuning, and evaluation design. Applied distributed computing to ensure the system scaled beyond in-memory limitations with a clear path to production deployment.

→

Outcome: Delivered a recommendation system showing meaningful improvement in personalization relevance over the popularity-baseline. Proved the architecture scalable using Spark's distributed framework, with a practical path to production deployment in a real e-commerce environment.

Project 03
Business Analytics · Real-World Application

Competitive Pricing Optimization Model

An Excel-based competitive pricing and margin analysis model built and applied at Maxx Group LLC — used to guide real product manufacturing and pricing decisions across multiple retail channels.

Excel Solver Data Modeling Pricing Analysis Competitive Intelligence Margin Analysis
Objective

To determine which products were viable to manufacture given competitor pricing across major retail channels, and to identify the optimal price points that protected margin while remaining competitive to end buyers.

Methods

Collected competitor pricing from Amazon, Walmart, Home Depot, and Lowe's. Built an Excel model mapping those prices against internal costs (factory-direct sourcing + container-volume purchasing). Defined minimum margin thresholds using Excel Solver. Calculated price differential opportunities and product-level viability flags for executive reporting.

My Role

Sole analyst. Collected all competitor data, built the pricing model from scratch, and reported findings directly to the business owner. Model outputs informed which products to list, discontinue, or reprice ahead of procurement decisions.

→

Outcome: The model identified product lines where competitor pricing undercut minimum viable margin — flagging them for discontinuation. For remaining products, it surfaced pricing windows that attracted buyers while preserving profitability. Findings reported directly to management and incorporated into product strategy.
